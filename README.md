# TrackSense Lab｜八路红外巡线算法实验室

面向高中生的浏览器本地仿真：同一二维赛道可切换分段控制与 PID 控制，便于比较控制思想与调参效果。

## 启动

```bash
npm install
npm run dev
# 生产构建验证
npm run build
```

## 模块与数据流

```
simulation/（赛道、传感器、电机、车辆、历史）
      SensorFrame ↓                         ↑ MotorCommand
controllers/（可插拔控制器及其运行上下文）
                         UI components/
```

`SimulationEngine` 只调用当前 `LineController.update(frame, parameters, context)`，不识别 PID 或分段控制。控制器返回 `ControllerResult`（给教学面板使用）及下一份私有运行上下文，再由引擎把左右目标速度交给电机模型。

传感器从 S1 到 S8 的权重为 `-3.5 ... 3.5`。黑线位置为所有检测到黑线的权重平均值；`00111100` 的位置为 `0`。`00000000` 是丢线，`11111111` 是全黑停止。

## 算法

- **分段控制**：按居中、轻微偏左/右、严重偏左/右选择固定轮速，入门直观，但轮速会跳变。
- **PID 控制**：以位置误差计算 P、I、D 和连续修正量；包含 dt 下限、积分与输出限幅、积分分离和丢线积分衰减。
- **手动轮速控制**：直接给左右轮设置速度，用来演示差速如何让固定小车视角下的赛道移动和旋转。

## 新增第三种算法

1. 在 `src/controllers/` 新建算法文件；
2. 实现 `LineController` 接口；
3. 定义 `parameterDefinitions` 参数 Schema 与预设；
4. 实现 `update()`，返回结果和下一份 context；
5. 提供伪代码、C 语言和 TypeScript 教学代码；
6. 注册到 `controllerRegistry`；
7. **无需修改 `simulationEngine`**，选择器、参数面板和代码区会自动出现新算法。

## 教学简化

本版本把小车视为固定视角，更新赛道的相对位置；电机以一阶响应连续趋近目标轮速，并叠加效率差与可选抖动，传感器采用宽度阈值与可选误判。它不模拟轮胎打滑、编码器、复杂交叉路口或真实红外反射率，适合讲清控制闭环而非替代实车标定。

## 推荐课堂流程

选择 S 形弯，先运行分段控制并观察轮速跳变；暂停且不重置场景，切换 PID，观察误差、P/I/D 和连续修正。再分别令 Ki 或 Kd 为 0，增大 Kp，最后开启轻微电机抖动比较抗干扰表现。
